RoomChat Project Documentation

System Architecture, Database Design, Sockets and Web API Guidelines

1. Technical Overview & Stack

RoomChat is an ephemeral, real-time workspace application designed to showcase full-stack development skills. It features zero user profiles, dynamic rooms, instant file sharing, and voice note recordings.

  • Frontend: React Single Page Application (SPA) structured using Vite, styling with custom light-theme Tailwind CSS configurations.
  • Backend: Node.js and Express server handling file streaming, socket handshakes, and room sessions.
  • Database: MongoDB Atlas instance for temporary message storage and binary file buffering.
  • Real-time Comm: Socket.IO managing participant lists, typing broadcasts, and instant message feeds.

2. Schema Model & Automatic Cleanup (TTL)

To eliminate data leaks, MongoDB collections employ Time-To-Live (TTL) indexes, automatically erasing documents 24 hours after creation.

A. Document Attachment Schema (documentSchema.js)

const documentSchema = new mongoose.Schema({
  fileName: { type: String, required: true },
  mimeType: { type: String, required: true },
  data: { type: Buffer, required: true }, // Files saved directly as binary data
  uploadDate: { type: Date, default: Date.now, index: { expires: 86400 } } // 24-Hour TTL
});

B. Message Schema (messageSchema.js)

const messageSchema = new mongoose.Schema({
  room: { type: String, required: true },
  name: { type: String, required: true },
  message: { type: String },
  kind: { type: String, enum: ['text', 'file'], default: 'text' },
  document: { type: mongoose.Schema.Types.ObjectId, ref: 'Document' },
  createdAt: { type: Date, default: Date.now, index: { expires: 86400 } } // 24-Hour TTL
});

3. Real-Time Socket Protocols

RoomChat handles real-time coordination with a series of high-frequency Socket.IO events:

  • join-room: Client joins a password-protected room. The server assigns the socket to an Express channel (socket.join(roomId)) and shares chat history.
  • active-users: Server broadcasts a list of online participants ({ name, socketId, tabId }) to the room header.
  • feedback: Broadcasts active typing statuses (e.g., "Jane is typing...") dynamically.
  • delete-message: Instantly deletes message entities from the database and broadcasts the deletion event.

4. Advanced Custom Features

A. Microphone Voice Notes Recorder

Built using the native MediaRecorder API, this tool prompts users for microphone permissions, captures audio streams, packages the stream chunks into a audio/webm File object, and sends it to the REST server using a multipart form request.

B. Custom Seekable Audio Player

Rather than using basic browser players, the audio player utilizes React state management hooks to handle media play/pause triggers and input timelines for an integrated custom UI.

C. Name-Hash Initial Avatars

Dynamic user avatars are generated directly on the frontend by taking the user's name initial and running a hashing function to match their profile with a high-contrast Tailwind color class, ensuring visual identification without any database storage.

5. Interview Talk Tracks & System Design Decisions

Use these responses during technical reviews to explain architectural trade-offs:

Why use MongoDB for file storage instead of Amazon S3?
"Since RoomChat is an ephemeral workspace application where all room sessions expire after 24 hours, storing file uploads as binary buffer data inside MongoDB simplifies our cleanups. We set a 24-hour TTL index directly on the document collections. When a room session expires, the database drops both the text messages and the binary file chunks in a single operation, preventing orphaned files and database storage leaks."
How did you handle the hover copy/delete buttons position?
"Initially, the floating actions (copy/delete) on right-aligned messages were positioned outside the bubble. However, the empty gap between the bubble and the actions caused the cursor to leave the container wrapper, immediately breaking the CSS hover state and closing the buttons before they could be clicked. I resolved this by adding left padding (pl-14) directly to the trigger container. This acts as an invisible bridge, keeping the mouse within the element boundary and ensuring a smooth click experience."
